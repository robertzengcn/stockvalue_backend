# Plan 02-01: Wire CacheManager into Application Lifecycle and Dependencies

## Goal
Initialize CacheManager in main.py lifespan, implement the get_cache dependency, and add cache configuration to ExternalDataConfig with all TTL policies.

## Tasks

### Task 1: Add Redis URL and additional TTLs to ExternalDataConfig
**File:** `stockvaluefinder/stockvaluefinder/config.py`
**What:**
- Add `REDIS_URL: str = "redis://localhost:6379/0"` field
- Add `RATE_CACHE_TTL: int = 3600` (1 hour for interest rates)
- Add `SHARES_CACHE_TTL: int = 86400` (24 hours)
- Add `DIVIDEND_CACHE_TTL: int = 86400` (24 hours)
- Add `FCF_CACHE_TTL: int = 86400` (24 hours)
- Add `CACHE_KEY_VERSION: str = "v1"` for cache key versioning

**Test:** Unit test that ExternalDataConfig is frozen and contains all TTL fields with correct defaults.

### Task 2: Initialize CacheManager in main.py lifespan
**File:** `stockvaluefinder/stockvaluefinder/main.py`
**What:**
- Import CacheManager and settings
- In lifespan startup: create CacheManager with REDIS_URL, call `cache.connect()`
- Store cache on `app.state.cache`
- In lifespan shutdown: call `cache.disconnect()`
- Wrap connect in try/except — log warning if Redis unavailable, continue without cache

**Test:** Unit test with mocked Redis verifying connect/disconnect called in lifespan.

### Task 3: Implement get_cache dependency
**File:** `stockvaluefinder/stockvaluefinder/api/dependencies.py`
**What:**
- Import CacheManager from utils.cache
- Add `_cache_instance: CacheManager | None = None` module-level variable
- Add `init_cache(redis_url: str) -> CacheManager` function called from lifespan
- Rewrite `get_cache()` to yield the module-level CacheManager instance
- If cache not initialized, yield None (graceful degradation)

**Test:** Unit test that get_cache yields None when not initialized, yields CacheManager when initialized.

### Task 4: Add cached_at timestamp to cache wrapper utility
**File:** `stockvaluefinder/stockvaluefinder/utils/cache.py`
**What:**
- Add `cacheable(prefix: str, ttl: int)` async context manager or wrapper function that:
  - Builds key: `{version}:{prefix}:{identifier}`
  - Checks cache, returns hit with `cached_at` metadata
  - On miss, calls underlying function, stores result with `cached_at` ISO timestamp
  - Returns `{**result, "_cache": {"hit": bool, "cached_at": str | None}}`
- Add helper `build_cache_key(version: str, prefix: str, *parts: str) -> str`

**Test:** Unit tests for build_cache_key and cacheable wrapper with mock Redis.
