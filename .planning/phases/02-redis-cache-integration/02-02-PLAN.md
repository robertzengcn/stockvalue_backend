# Plan 02-02: Cache ExternalDataService Methods and Wire into Routes

## Goal
Wrap all ExternalDataService data-fetching methods with Redis caching using appropriate TTLs. Each cached response includes a `cached_at` timestamp. Route responses indicate cache status.

## Tasks

### Task 1: Add CacheManager to ExternalDataService and cache financial_report
**File:** `stockvaluefinder/stockvaluefinder/external/data_service.py`
**What:**
- Add `cache: CacheManager | None = None` parameter to `__init__`
- Add `cache_version: str = "v1"` parameter to `__init__`
- In `get_financial_report`: before fallback chain, check cache with key `v1:fin_report:{ticker}:{year}`
- On cache hit: return cached result (skip all API calls)
- On cache miss: proceed with existing fallback chain, then cache result with TTL=86400
- Same pattern for `get_financial_report` — use `_is_development_mode()` bypass

**Test:** Unit test with mock CacheManager verifying cache hit returns early, cache miss stores result.

### Task 2: Cache remaining data service methods
**File:** `stockvaluefinder/stockvaluefinder/external/data_service.py`
**What:**
- `get_current_price` → key `v1:price:{ticker}`, TTL=300
- `get_shares_outstanding` → key `v1:shares:{ticker}`, TTL=86400
- `get_free_cash_flow` → key `v1:fcf:{ticker}:{period}`, TTL=86400
- `get_dividend_yield` → key `v1:div_yield:{ticker}`, TTL=86400
- `get_stock_basic` → key `v1:stock_basic:{ts_code}`, TTL=604800
- For each: check cache first, on miss fetch + store, skip in dev mode
- Helper method `_cache_get_or_set(key, ttl, fetch_fn)` to DRY the pattern

**Test:** Unit tests for each cached method with mock CacheManager verifying correct keys and TTLs.

### Task 3: Wire CacheManager into dependencies and pass to data_service
**File:** `stockvaluefinder/stockvaluefinder/api/dependencies.py`
**What:**
- Update `get_data_service()` to accept optional `cache: CacheManager | None`
- Update `get_initialized_data_service()` to get cache from `get_cache()` and pass to service
- Ensure cache=None works (no caching when Redis unavailable)

**Test:** Integration test verifying data_service receives cache instance through DI.
