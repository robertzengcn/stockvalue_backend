Add NEW test classes to the END of test_data_service.py. Do NOT modify any existing tests.

Create these test classes:

3. **TestGetSharesOutstanding** class:

4. **TestGetDividendYield** class:
   - `test_returns_yield_as_float`: Mock data source. Assert get_dividend_yield("600519.SH") returns a float between 0 and 1.

5. **TestGetStockBasic** class:
   - `test_returns_stock_metadata`: Call get_stock_basic("600519.SH"). Assert result dict contains ticker, name, market fields.

6. **TestFallbackChain** class:
   - `test_akshare_primary_efinance_secondary`: Create service with akshare+efinance enabled. Mock akshare to return None (or raise), mock efinance to return valid data. Call get_financial_report("600519.SH", 2023). Assert efinance data returned.
   - `test_all_sources_fail_uses_mock`: Create service with all real sources mocked to fail, in development mode. Assert mock data is returned.

7. **TestFieldNormalization** class:
   - `test_akshare_fields_mapped_to_standard_keys`: If _get_financial_report_from_akshare or similar normalization method exists, call it with mock AKShare data and verify the output dict uses standardized English keys (revenue, net_income, operating_cash_flow, etc.) not Chinese AKShare field names.

Read the data_service.py source carefully to understand the exact method signatures, return types, and fallback logic before writing tests. The key pattern is: create service, mock individual _fetch_* methods or client methods, call public method, assert result.
